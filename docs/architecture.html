Thought Box v1 · architecture

Dictation-first thought collector. Kotlin + Ktor backend on Google Cloud Run, Next.js PWA on Vercel, Neon Postgres, AWS S3 for audio, OpenAI Whisper + gpt-4o-mini for STT and enrichment. Single-user v1, schema is multi-user-ready.

1 · System topology

One request from "tap record" to "stored, transcribed, enriched thought" spans four external services and one async pipeline. The PWA only talks to the backend; everything else is server-to-server.

request flow in-process composition deployment boundary

2 · The async pipeline

Upload is fast; transcription and enrichment run on an in-process coroutine worker. Status is one column on thoughts. The client polls a single endpoint until the status is terminal.

success / progress failure / retries exhausted recovery on boot

Retry policy: 3 attempts, exponential backoff 1s · 4s · 16s. attempts, last_error, last_attempt_at live on the row — no separate retry table. Idempotency-Key is set on chat completions (derived from thought_id + prompt_version); Whisper does not support it, so a retry could double-bill — accepted at side-project cost.

3 · Module map

Deep modules behind small interfaces. Every external service hides behind one Kotlin interface with exactly one v1 implementation. Single swap point per provider.

Config

env → typed object

Single source of truth, validated at boot. No other module reads env directly.

Transcriber

interface · OpenAiTranscriber

Hides Whisper HTTP, response parsing, error categorization. Hand-rolled via Ktor HttpClient — no official SDK.

Enricher

interface · OpenAiEnricher

JSON-schema chat completions. Returns category + tags + title + summary. Idempotency-Key per (thought, prompt_version).

BlobStore

interface · S3BlobStore

put / get / exists over byte streams. S3_ENDPOINT override switches MinIO ↔ S3 with zero if isDev.

ThoughtRepository

kotliquery · raw SQL

Inserts, status transitions, cursor pagination, sweep queries. Domain types out, SQL strings in.

UserResolver

single swap point

v1: hardcoded seeded UUID. Future: JWT extractor. Every query routes through this for current_user_id.

ThoughtPipeline

coroutine worker

The heart. Reads pending, drives the state machine, owns retry + backoff, updates attempts and last_error.

StartupRecovery

boot sweep

Reclaims rows stuck in transcribing / enriching after a crash or deploy.

HTTP routes

thin handlers

Ktor route handlers parse, dispatch, render. Zero business logic — that lives in pipeline + repository.

5 · HTTP surface

Six endpoints. JSON. Cursor pagination. No GraphQL, no auth in v1.

MethodPathPurpose
POST /thoughts Multipart upload → stream to S3 → insert pending row → return { id, status, created_at }
GET /thoughts/:id Polling endpoint. Returns status, transcript (if any), enrichment (if any), audio metadata
GET /thoughts Cursor-paginated list. Optional category and tag filters
GET /thoughts/:id/audio Streams original audio bytes with stored Content-Type
GET /config Returns { max_duration_ms, min_duration_ms, max_size_bytes } so PWA never hardcodes limits
GET /healthz Liveness probe + PWA keepalive ping on mount (warms Cloud Run + Neon)

6 · Environment isolation

Dev and prod share no resources. Same code path, different names. Misconfiguration crashes loudly instead of writing to the wrong store.

dev (local laptop)

docker compose · localhost

Postgres in Compose, MinIO in Compose, dev OpenAI key, dev IAM keys scoped to thoughts-dev. S3_ENDPOINT=http://localhost:9000.

prod (Cloud Run)

*.run.app · managed

Neon Postgres, AWS S3 (thoughts-prod), prod OpenAI key, prod IAM keys. S3_ENDPOINT unset → defaults to AWS.

Guardrails

different resource names

Distinct bucket and database names per env. Wrong creds → bucket doesn't exist → app crashes loudly.

no prod creds on laptop

Prod IAM keys live only in GH Secrets + Cloud Run env. Laptop only has dev-scoped IAM.

startup banner

App logs the active env + Postgres host + S3 endpoint at boot. Cheap, prevents "which DB am I on?" confusion.

8 · Delivery roadmap

Seven vertical-slice tracer bullets. Each is end-to-end demoable. HITL needs user setup (credentials, accounts); AFK runs unattended.

01 · Skeleton deploys HITL
Ktor /healthz + Dockerfile + Cloud Run + Vercel Next.js + CI green + manual-deploy workflow. PWA pings /healthz on mount.
02 · DB + config + seeded user HITL
Neon, Flyway, users table, Config, UserResolver, env banner. GET /me returns seeded UUID.
03 · Audio upload + playback HITL
S3 + MinIO via BlobStore. thoughts table. PWA records, uploads, replays. No AI yet.
04 · Transcription pipeline HITL
Transcriber + OpenAiTranscriber. Pipeline worker, retry policy, recovery sweep. PWA polls and renders transcript.
05 · Enrichment + detail view AFK
Enricher + JSON-schema chat completions. thought_enrichments table. Detail view renders title + summary + category + tags.
06 · List + filters AFK
Cursor-paginated GET /thoughts, category + tag filters. PWA list view + filter chips.
07 · Observability AFK
Logback JSON to stdout, structured events at request + transition + external-call boundaries, Sentry on uncaught exceptions.

9 · Cost ceiling (free-tier accounting)

Cloud Run

2M req · 360k GB-s / mo

Free forever at this scale. Cold start ~3–5s after idle, mitigated by PWA keepalive.

Neon

0.5 GB · 190 compute-hr / mo

Free forever. Scale-to-zero compounds cold start; pooled-mode wake ~1s.

Vercel

Hobby tier

Frontend static + serverless. Free, no budget worries at side-project volume.

AWS S3

5 GB free for 12 mo

Audio storage is < 1 GB in realistic use. After year 1, ~$0.02–0.10/mo. Effectively free.

OpenAI

pay per use

Whisper $0.006/min · gpt-4o-mini negligible. 180 one-min thoughts/mo ≈ $1.10. Capped by per-key usage limits.

Sentry

5k events / mo

Free indefinitely at this scale.

GitHub Actions

2000 min / mo private

CI + deploy combined run well under the cap.

Domain

deferred

v1 uses Cloud Run's *.run.app + Vercel's *.vercel.app. No domain purchase.
